TypeScript Tips for Better Code

Advanced TypeScript patterns and techniques that will help you write more maintainable and type-safe code. From utility types to conditional types.

TypeScript JavaScript Code Quality

TypeScript has evolved from a simple type checker to a powerful language that enables developers to write more reliable and maintainable code. After years of working with TypeScript in production, I've discovered patterns and techniques that significantly improve code quality and developer experience.

Utility Types Mastery

TypeScript's built-in utility types are incredibly powerful, yet many developers only scratch the surface. Here are some advanced patterns:

Pick and Omit

interface User {
  id: string;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
}

// Create a public user type without sensitive data
type PublicUser = Omit;

// Create a user creation payload
type CreateUserPayload = Pick;

// Even better: combine with conditional logic
type UserResponse = T extends true 
  ? User 
  : PublicUser;

Partial and Required

// Make all properties optional for updates
type UpdateUserPayload = Partial;

// Make specific optional properties required
type UserWithRequiredEmail = Required> & 
                             Partial>;

Advanced Type Patterns

Branded Types

Create types that are structurally identical but logically distinct: 

type UserId = string & { readonly brand: unique symbol };
type PostId = string & { readonly brand: unique symbol };

const createUserId = (id: string): UserId => id as UserId;
const createPostId = (id: string): PostId => id as PostId;

// This prevents accidental mixing of IDs
function getUser(userId: UserId) { /* ... */ }
function getPost(postId: PostId) { /* ... */ }

const userId = createUserId('user-123');
const postId = createPostId('post-456');

getUser(postId); // TypeScript error!

Template Literal Types

Create powerful string patterns with type safety: 

type EventType = 'click' | 'hover' | 'focus';
type ElementType = 'button' | 'input' | 'div';

// Generate all possible event handler names
type EventHandler = `on${Capitalize}`;

// Even more advanced: generate CSS selectors
type CSSSelector = 
  | T 
  | `.${string}` 
  | `#${string}` 
  | `${T}.${string}`;

type ButtonSelector = CSSSelector<'button'>; 
// 'button' | '.${string}' | '#${string}' | 'button.${string}'

Conditional Types Deep Dive

Conditional types enable sophisticated type logic: 

// Extract array element type
type ArrayElement = T extends (infer U)[] ? U : never;

type StringArray = string[];
type ElementType = ArrayElement; // string

// Create a deep readonly type
type DeepReadonly = {
  readonly [P in keyof T]: T[P] extends object 
    ? DeepReadonly 
    : T[P];
};

// Function return type extraction
type ReturnType = T extends (...args: any[]) => infer R ? R : never;

// Async function return type
type AsyncReturnType = T extends (...args: any[]) => Promise 
  ? R 
  : never;

Generic Constraints

Use constraints to create more expressive and safe generic functions: 

// Constrain to objects with specific properties
function updateEntity(
  entity: T, 
  updates: Partial>
): T {
  return { ...entity, ...updates };
}

// Constrain to specific keys
function pluck(obj: T, keys: K[]): T[K][] {
  return keys.map(key => obj[key]);
}

// Usage
const user = { id: '1', name: 'John', email: 'john@example.com' };
const values = pluck(user, ['name', 'email']); // string[]

Mapped Types

Create types that transform existing types systematically: 

// Make all properties optional except specified ones
type PartialExcept = Partial & Pick;

// Create event handlers from event names
type EventHandlers> = {
  [K in keyof T as `on${Capitalize}`]: (
    event: T[K]
  ) => void;
};

type Events = {
  click: MouseEvent;
  keypress: KeyboardEvent;
};

type Handlers = EventHandlers;

Error Handling Patterns

TypeScript enables powerful error handling patterns: 

// Result type for better error handling
type Result = 
  | { success: true; data: T }
  | { success: false; error: E };

async function fetchUser(id: string): Promise> {
  try {
    const user = await api.getUser(id);
    return { success: true, data: user };
  } catch (error) {
    return { success: false, error: error as Error };
  }
}

// Usage with type narrowing
const result = await fetchUser('123');
if (result.success) {
  console.log(result.data.name); // TypeScript knows this is User
} else {
  console.error(result.error.message); // TypeScript knows this is Error
}

Performance Considerations

TypeScript compilation performance matters in large codebases:

Optimize Import Types

// Use type-only imports when possible
import type { User } from './types';
import { validateUser } from './validation';

// This helps with tree-shaking and compilation speed

Avoid Complex Types

// Instead of deeply nested conditional types
type Complex = T extends A ? B extends C ? D : E : F;

// Break them down
type StepOne = T extends A ? B : never;
type StepTwo = StepOne extends C ? D : E;
type Simple = T extends A ? StepTwo : F;

Testing with Types

Ensure your types work correctly with compile-time tests: 

// Type assertion tests
type Equals = (() => T extends X ? 1 : 2) extends 
                    (() => T extends Y ? 1 : 2) ? true : false;

// Test your utility types
type Test1 = Equals, string>; // true
type Test2 = Equals, 
                   { readonly a: { readonly b: number } }>; // true

// Use expectType utility
declare function expectType(value: T): void;

const user: User = { /* ... */ };
expectType(omit(user, 'password'));

Configuration Best Practices

Your tsconfig.json significantly impacts your TypeScript experience: 

{
  "compilerOptions": {
    "strict": true,                    // Enable all strict checks
    "noUncheckedIndexedAccess": true, // Safer array/object access
    "exactOptionalPropertyTypes": true, // Stricter optional properties
    "noImplicitReturns": true,        // Ensure all code paths return
    "noFallthroughCasesInSwitch": true, // Safer switch statements
    "noImplicitOverride": true,       // Explicit override keyword
    "allowUnusedLabels": false,       // Catch unused labels
    "allowUnreachableCode": false     // Catch unreachable code
  }
}
